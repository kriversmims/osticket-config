<h1>osTicket - Post-Install Configuration</h1>
This repository outlines the post-install configuration of the open-source help desk ticketing system osTicket.<br />

<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10</b> (21H2)

<h2>Post-Install Configuration Objectives</h2>

- Roles
- Departments
- Teams
- Agents
- Users
- SLA's
- Help Topics

<p>
<b>Roles</b> Roles in osTicket define what actions a staff member can perform within the helpdesk system. They are tied to departments and help establish permission control for managing tickets, users, and internal workflows. In this instance, I created a new role called "Supreme Admin" and gave this role all permissions.
